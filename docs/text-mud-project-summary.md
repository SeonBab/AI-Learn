# TextMUDGame 프로젝트 요약

## 설명

`TextMUDGame`은 C++ 학습을 위해 만든 콘솔 기반 텍스트 MUD 게임 프로젝트이다. 목표는 거대한 온라인 MUD를 만드는 것이 아니라, 작은 싱글플레이 텍스트 RPG를 구현하면서 C++의 객체 설계, 명령어 파싱, 상태 관리, 파일 입출력, 로컬 LLM 연동 구조를 단계적으로 익히는 것이다.

게임의 기본 시나리오는 `낡은 성문의 비밀`이다. 플레이어는 작은 마을에서 시작해 주변 장소를 탐험하고, 아이템을 얻고, NPC와 대화하며, 잠긴 성문을 여는 목표를 따라간다. 현재 구현은 콘솔에 영어 명령어를 입력하고, 결과 설명은 한글로 출력하는 방식이다.

기본 플레이 흐름은 다음과 같다.

```text
명령어 입력
-> CommandParser가 입력 해석
-> GameEngine이 게임 규칙 처리
-> World, Player, GameState 변경 또는 조회
-> 결과 문자열 반환
-> 콘솔에 출력
```

현재 주요 명령어는 다음과 같다.

| 명령어 | 역할 |
|---|---|
| `look` | 현재 장소 설명을 확인한다. |
| `go` | 다른 장소로 이동한다. |
| `items` | 현재 장소의 아이템을 확인한다. |
| `take` | 아이템을 획득한다. |
| `use` | 특정 아이템을 사용한다. |
| `inventory` | 플레이어 인벤토리를 확인한다. |
| `talk` | NPC의 고정 대사를 확인한다. |
| `ask` | 로컬 LLM을 통해 NPC에게 자연어 질문을 한다. |
| `save` | 현재 진행 상태를 파일로 저장한다. |
| `load` | 저장된 진행 상태를 불러온다. |
| `help` | 사용 가능한 명령어를 확인한다. |
| `quit` | 게임을 종료한다. |

프로젝트는 역할별 클래스로 나뉘어 있다.

| 클래스 | 역할 |
|---|---|
| `GameEngine` | 게임 진행의 중심이며 명령어 처리와 결과 문자열 생성을 담당한다. |
| `CommandParser` | 사용자 입력 문자열을 명령어와 인자로 변환한다. |
| `World` | 방, 아이템, NPC 같은 고정 월드 데이터를 관리한다. |
| `Room` | 장소 이름, 설명, 이동 경로, 아이템, NPC 정보를 가진다. |
| `Item` | 아이템 ID, 이름, 설명, 사용 효과를 표현한다. |
| `NPC` | NPC ID, 이름, 고정 대사, AI 응답 설정을 표현한다. |
| `Player` | 현재 위치와 인벤토리 같은 플레이어 상태를 관리한다. |
| `GameState` | 문 잠금 해제, 퀘스트 진행 등 전체 진행 상태를 관리한다. |
| `SaveManager` | 게임 상태를 파일로 저장하고 다시 불러오는 기능을 담당한다. |
| `LLMClient` | Ollama 같은 로컬 LLM 호출을 담당한다. |

AI 기능은 `ask` 명령어와 연결된다. 플레이어가 NPC에게 질문하면 `GameEngine`이 현재 방, NPC 정보, 플레이어 인벤토리, 게임 진행 상태를 바탕으로 프롬프트를 만들고, `LLMClient`가 로컬 Ollama 모델 호출을 시도한다. Ollama 서버나 모델을 사용할 수 없는 경우에도 게임이 종료되지 않도록 fallback 응답을 제공한다.

저장 기능은 학습용으로 단순한 텍스트 파일 기반 구조에서 출발한다. 현재 위치, 인벤토리, 성문 잠금 해제 여부, 퀘스트 진행 상태처럼 게임을 이어서 플레이하는 데 필요한 정보를 저장 대상으로 삼는다.

이 프로젝트의 학습 가치는 기능 자체보다 구조에 있다. 명령어 입력과 게임 규칙 처리를 분리하고, 월드 데이터와 플레이어 상태를 나누며, 로컬 LLM은 직접 게임 상태를 바꾸지 않고 NPC 대화와 힌트 생성에만 참여하도록 설계한다. 이를 통해 C++ 콘솔 프로그램을 작은 객체 지향 게임 구조로 확장하는 과정을 연습할 수 있다.

## 핵심 정리

- `TextMUDGame`은 C++ 학습용 콘솔 기반 텍스트 MUD 게임이다.
- 기본 시나리오는 `낡은 성문의 비밀`이며, 탐험, 이동, 아이템, NPC 대화, 저장과 불러오기를 다룬다.
- 입력은 `CommandParser`, 게임 규칙은 `GameEngine`, 월드 데이터는 `World`, 플레이어 상태는 `Player`가 담당한다.
- `ask` 명령어는 `LLMClient`를 통해 로컬 Ollama 모델과 연결된다.
- 로컬 LLM은 NPC 대화와 힌트 생성을 담당하며, 실제 게임 상태 변경은 C++ 게임 로직이 처리한다.
- 저장과 불러오기는 `SaveManager`를 통해 관리한다.
- 프로젝트의 핵심 학습 목표는 C++ 객체 설계, 명령어 파싱, 상태 관리, 파일 입출력, 로컬 AI 연동 구조를 함께 익히는 것이다.
