# 텍스트 MUD 첫 번째 플레이 루프와 시나리오

## 설명

이 문서는 MUD 게임을 바로 코딩하기 전에 가장 먼저 고정해야 할 최소 플레이 루프와 첫 번째 플레이 가능한 시나리오를 정리한다.

현재 프로젝트의 두 번째 목표는 1단계 로컬 LLM 채팅 프로그램을 바탕으로 텍스트 기반 MUD 게임을 기획하고 제작하는 것이다. 따라서 처음부터 큰 RPG 시스템을 만들지 않고, 플레이어가 반복하게 될 가장 작은 경험부터 설계한다.

## 핵심 정리

1. 첫 번째 작업은 `이 MUD 게임의 첫 번째 플레이어 경험은 무엇인가?`에 답하는 것이다.
2. 첫 버전의 핵심 반복은 `장소 확인 -> 명령어 입력 -> 명령어 해석 -> 결과 출력 -> 다시 명령어 입력`이다.
3. 첫 번째 시나리오는 `마을 입구 -> 숲 -> 상점 -> NPC 대화 -> 아이템 획득 -> NPC에게 전달` 흐름으로 잡는다.
4. 로컬 LLM NPC는 첫 버전에서 게임 상태를 직접 바꾸지 않고, 자연스러운 대화와 힌트 제공만 담당한다.
5. 구현 전에 방 구조, 명령어 목록, 아이템, NPC, AI 응답 범위를 이 문서 기준으로 설계한다.

## 첫 번째 플레이어 경험

이 MUD 게임의 첫 번째 플레이어 경험은 다음과 같다.

```text
플레이어는 작은 마을 입구에서 시작한다.
주변 장소를 살펴보고, 북쪽의 숲이나 동쪽의 상점으로 이동할 수 있다.
마을 사람 NPC와 대화할 수 있으며, 일부 NPC는 로컬 LLM을 통해 자연스럽게 응답한다.
첫 번째 목표는 숲에 떨어진 아이템을 찾아 마을 사람에게 가져다주는 것이다.
```

이 경험은 다음 학습 목표를 만족한다.

| 학습 요소 | 플레이 경험에서의 역할 |
|---|---|
| 장소 설명 | `look` 명령어로 현재 방 정보를 출력한다. |
| 명령어 파싱 | `north`, `east`, `talk npc` 같은 입력을 해석한다. |
| 방 이동 | 플레이어의 현재 위치를 변경한다. |
| NPC 확인 | 현재 장소에 있는 NPC를 보여준다. |
| NPC 대화 | 고정 대사 또는 로컬 LLM 응답을 출력한다. |
| 아이템 획득 | 특정 장소의 아이템을 인벤토리에 넣는다. |
| 간단한 목표 달성 | 아이템을 가져다주면 첫 번째 목표가 완료된다. |

## 최소 플레이 루프

첫 버전의 최소 플레이 루프는 다음과 같다.

```text
플레이어가 현재 장소 설명을 본다
-> 명령어를 입력한다
-> 게임이 명령어를 해석한다
-> 장소 이동, 아이템 확인, NPC 대화 같은 결과가 출력된다
-> 다시 명령어를 입력한다
```

코드 구조로 바꾸면 다음 흐름이 된다.

```text
while game is running:
    현재 상태 출력 또는 프롬프트 출력
    사용자 입력 받기
    CommandParser가 명령어 해석
    GameEngine이 현재 상태를 기준으로 명령 처리
    결과 메시지를 문자열로 반환
    콘솔에 결과 출력
```

이 구조를 따르면 나중에 콘솔 입력을 네트워크 입력으로 바꿔도 `GameEngine`의 핵심 로직을 유지할 수 있다.

## 첫 번째 버전의 명령어 범위

첫 playable 버전에서는 다음 명령어만 구현한다.

| 명령어 | 설명 | 예시 |
|---|---|---|
| `look` | 현재 장소 설명을 본다. | `look` |
| `north` | 북쪽 방으로 이동한다. | `north` |
| `south` | 남쪽 방으로 이동한다. | `south` |
| `east` | 동쪽 방으로 이동한다. | `east` |
| `west` | 서쪽 방으로 이동한다. | `west` |
| `npc` | 현재 장소의 NPC를 확인한다. | `npc` |
| `talk npc` | 현재 장소의 NPC와 대화한다. | `talk villager` |
| `items` | 현재 장소의 아이템을 확인한다. | `items` |
| `take item` | 현재 장소의 아이템을 줍는다. | `take herb` |
| `inventory` | 인벤토리를 확인한다. | `inventory` |
| `give item npc` | 아이템을 NPC에게 전달한다. | `give herb villager` |
| `quit` | 게임을 종료한다. | `quit` |

기존 전체 기획 문서에는 `go north`, `npcs`, `ask npc 질문` 같은 확장 명령어가 포함되어 있다. 첫 번째 플레이 루프에서는 입력 부담을 줄이기 위해 `north`, `east`, `npc`처럼 짧은 명령어를 우선 사용한다.

## 첫 번째 플레이 가능한 시나리오

첫 번째 시나리오 제목은 임시로 `숲길의 작은 약초`로 정한다.

### 시나리오 목표

```text
플레이어는 마을 입구에서 시작한다.
마을 사람에게 말을 걸어 약초가 필요하다는 힌트를 얻는다.
북쪽 숲길로 이동해 작은 약초를 발견한다.
약초를 주워 마을 입구로 돌아온다.
마을 사람에게 약초를 전달하면 첫 번째 목표가 완료된다.
```

### 기본 진행 흐름

```text
look
npc
talk villager
north
look
items
take herb
south
give herb villager
quit
```

### 선택 흐름

상점은 첫 버전에서 필수 목표는 아니지만, 플레이어가 이동과 NPC 대화를 한 번 더 실험할 수 있는 장소로 둔다.

```text
east
look
npc
talk merchant
west
```

## 방 구조

첫 번째 시나리오의 방은 3개로 제한한다.

```text
          숲길
            |
마을 입구 -- 상점
```

| 방 ID | 방 이름 | 설명 | 이동 가능 방향 | 아이템 | NPC |
|---|---|---|---|---|---|
| `village_entrance` | 마을 입구 | 작은 마을로 들어가는 흙길이다. 북쪽에는 숲길이 있고, 동쪽에는 작은 상점이 있다. | `north`, `east` | 없음 | `villager` |
| `forest_path` | 숲길 | 키 낮은 나무와 풀숲이 이어진 조용한 숲길이다. 땅 위에 작은 약초가 보인다. | `south` | `herb` | 없음 |
| `shop` | 상점 | 나무 진열대와 오래된 계산대가 있는 작은 상점이다. | `west` | 없음 | `merchant` |

## 아이템

첫 번째 시나리오의 아이템은 1개만 사용한다.

| 아이템 ID | 이름 | 설명 | 사용 목적 |
|---|---|---|---|
| `herb` | 작은 약초 | 숲길 풀숲 사이에서 발견한 향이 강한 약초다. | 마을 사람에게 전달해 첫 목표를 완료한다. |

첫 버전에서는 아이템 사용 명령어보다 `give herb villager`를 우선 구현한다. `use item`은 잠긴 문, 장치, 회복 아이템이 생긴 뒤 추가한다.

## NPC

첫 번째 시나리오의 NPC는 2명으로 제한한다.

| NPC ID | 이름 | 위치 | 응답 방식 | 역할 |
|---|---|---|---|---|
| `villager` | 마을 사람 | 마을 입구 | 고정 대사 또는 로컬 LLM | 첫 목표를 안내하고 약초를 받는다. |
| `merchant` | 상인 | 상점 | 고정 대사 | 상점 위치와 마을 분위기를 설명한다. |

## AI NPC 응답 범위

로컬 LLM을 사용하는 NPC는 첫 버전에서 `villager` 한 명으로 제한한다.

AI NPC는 다음 역할만 수행한다.

1. 약초가 필요하다는 힌트를 자연스럽게 말한다.
2. 숲길 방향을 간접적으로 알려준다.
3. 마을의 분위기나 상황을 짧게 설명한다.
4. 플레이어가 이미 약초를 가지고 있는지에 따라 대화 톤을 조금 바꾼다.

AI NPC는 다음 행동을 하지 않는다.

1. 아이템을 직접 생성하지 않는다.
2. 인벤토리를 직접 변경하지 않는다.
3. 목표 완료 상태를 직접 변경하지 않는다.
4. 새로운 장소, 아이템, NPC를 즉석에서 만들지 않는다.

목표 완료는 반드시 `give herb villager` 명령을 처리하는 C++ 게임 로직이 담당한다.

## 첫 번째 목표 완료 조건

첫 번째 목표 완료 조건은 다음과 같다.

```text
현재 위치가 village_entrance이다.
플레이어 인벤토리에 herb가 있다.
플레이어가 give herb villager 명령어를 입력한다.
```

완료 시 게임은 다음 상태를 변경한다.

```text
inventory에서 herb 제거
quest_stage를 completed로 변경
완료 메시지 출력
```

예시 출력은 다음과 같다.

```text
마을 사람은 작은 약초를 조심스럽게 받아 들었다.
"고맙네. 덕분에 오늘 밤은 한숨 돌릴 수 있겠어."

첫 번째 목표를 완료했다.
```

## 애매하거나 결정이 필요한 부분

현재 단계에서 애매한 부분은 다음과 같다.

1. 이동 명령어를 `north`로 할지 `go north`로 할지 결정해야 한다.
2. NPC 확인 명령어를 `npc`로 할지 `npcs`로 할지 결정해야 한다.
3. AI NPC 질문 명령어를 첫 버전에 포함할지, 고정 대사 이후에 붙일지 결정해야 한다.
4. 아이템 전달 명령어 `give herb villager`를 첫 버전에 넣으면 퀘스트 완료 흐름이 명확하지만, 명령어 파서가 조금 더 복잡해진다.

이번 첫 플레이 루프 문서에서는 사용자가 제시한 흐름에 맞춰 `north`, `south`, `east`, `west`, `npc`, `talk npc`를 우선 채택한다. 다만 기존 기획 문서와의 호환을 위해 나중에 `go north`, `npcs`를 별칭으로 추가할 수 있다.

## 다음 작업

이 문서를 기준으로 다음 작업을 진행한다.

1. `Room`, `Item`, `NPC`, `Player`, `GameState`에 필요한 최소 필드를 정한다.
2. `CommandParser`가 처리할 명령어 형태를 확정한다.
3. `GameEngine::processCommand`의 입력과 출력 형식을 정한다.
4. `TextMUDGame` 프로젝트에 첫 플레이 루프를 구현한다.
5. 로컬 LLM NPC를 붙이기 전에 고정 대사 버전으로 전체 흐름을 먼저 완성한다.
