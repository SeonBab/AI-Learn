# 텍스트 MUD 첫 플레이 루프 설계 작업 기록

## 설명

사용자는 MUD 게임의 첫 작업으로 바로 코딩하지 말고, 게임의 최소 플레이 루프와 첫 번째 플레이 가능한 시나리오를 먼저 정해야 한다고 지시했다.

이번 작업에서는 플레이어가 처음 반복하게 될 경험, 최소 명령어 목록, 첫 번째 방 구조, NPC, 아이템, AI 응답 범위, 목표 완료 조건을 문서로 정리했다.

## 핵심 정리

## 사용자가 지시한 내용

1. MUD 게임에서 제일 처음 해야 할 작업은 게임의 최소 플레이 루프를 정하는 것이다.
2. `이 MUD 게임의 첫 번째 플레이어 경험은 무엇인가?`에 답해야 한다.
3. 첫 번째 플레이 가능한 MUD 시나리오 1개를 만든다.
4. 필요하거나 애매하다고 판단되는 작업이 있으면 알려야 한다.

## assistant가 작업한 내용

1. 기존 `docs/text-mud-game-plan.md`의 전체 기획과 겹치지 않도록 첫 플레이 루프 전용 문서를 분리했다.
2. 첫 번째 플레이어 경험을 `마을 입구 -> 숲길 -> 상점 -> NPC 대화 -> 아이템 획득 -> NPC에게 전달` 흐름으로 정리했다.
3. 첫 번째 시나리오 제목을 `숲길의 작은 약초`로 잡았다.
4. 첫 버전의 명령어 범위를 `look`, `north`, `south`, `east`, `west`, `npc`, `talk npc`, `items`, `take item`, `inventory`, `give item npc`, `quit`로 정리했다.
5. 방 3개, 아이템 1개, NPC 2명의 최소 구성을 설계했다.
6. 로컬 LLM을 사용하는 NPC는 첫 버전에서 `villager` 한 명으로 제한했다.
7. 애매하거나 결정이 필요한 부분을 문서에 별도로 남겼다.
8. README 문서 목록에 첫 플레이 루프 문서를 추가했다.

## assistant가 답변한 핵심 내용

1. 첫 번째 플레이어 경험은 작은 마을 입구에서 시작해 숲에서 약초를 찾고 마을 사람에게 전달하는 흐름이다.
2. 첫 번째 플레이 루프는 장소 확인, 명령어 입력, 명령어 해석, 결과 출력, 재입력의 반복이다.
3. 첫 버전은 고정 대사 기반으로 전체 흐름을 먼저 완성하고, 이후 로컬 LLM NPC를 붙이는 것이 좋다.
4. `north`와 `go north`, `npc`와 `npcs` 같은 명령어 표기는 아직 결정이 필요한 부분이다.
