# 상황별 답변 형식과 로딩 메시지 작업 기록

## 설명

사용자는 `안녕` 같은 단순 인사에 C++ 학습 노트 형식의 긴 답변이 나오는 것은 불필요하다고 지적했다.

또한 사용자가 채팅을 보낸 뒤 AI가 답변하기까지 로딩 중임을 알려주는 표시가 필요하다고 요청했다.

이에 따라 단순 인사는 짧게 답하고, C++ 학습 설명이 필요한 질문에만 Markdown 학습 노트 형식을 사용하도록 프롬프트와 응답 흐름을 수정했다.

## 사용자 지시 내용

1. 단순 인사에는 긴 C++ 학습 답변을 하지 않는다.
2. `## 설명`, `## 핵심 정리`, 코드 예시는 필요한 질문에만 사용한다.
3. AI 응답 생성 중임을 알 수 있는 로딩 메시지를 추가한다.

## assistant가 작업한 내용

1. `AIEngine` 프롬프트를 상황별 답변 방식으로 수정했다.
2. 단순 인사에는 예시, 목록, 코드, 학습 안내를 붙이지 않도록 지시를 강화했다.
3. `안녕`, `안녕하세요`, `하이`, `hi`, `hello` 같은 단순 인사는 모델 호출 전에 짧게 응답하도록 처리했다.
4. AI 응답 생성 직전에 `AI 답변 생성 중...` 메시지를 출력하도록 했다.
5. `Debug | x64` 구성을 MSBuild로 빌드해 성공 여부를 확인했다.
6. `hi` 입력에 `안녕하세요.`로 짧게 응답하는 것을 확인했다.

## assistant가 답변한 핵심 내용

이제 단순 인사는 짧게 응답하고, C++ 개념 설명이나 코드 작성 같은 요청일 때만 학습 노트 형식의 Markdown 답변을 사용한다.

AI 응답을 기다리는 동안에는 `AI 답변 생성 중...` 메시지가 출력된다.

## 생성하거나 수정한 파일

| 파일 | 내용 |
|---|---|
| `LocalAIChat/src/AIEngine.cpp` | 상황별 답변 프롬프트와 단순 인사 응답 처리 추가 |
| `LocalAIChat/src/ChatApp.cpp` | AI 응답 생성 전 로딩 메시지 출력 추가 |
| `docs/work-logs/context-aware-response-and-loading.md` | 이번 작업 기록 |

## 확인 결과

다음 구성으로 빌드 확인을 완료했다.

```text
Configuration=Debug
Platform=x64
```

MSBuild 결과는 경고 0개, 오류 0개이다.

실행 확인 결과는 다음과 같다.

```text
User: hi
AI 답변 생성 중...
AI: 안녕하세요.
```

## 다음에 이어서 할 수 있는 작업

1. 로딩 메시지를 한 줄에서 갱신되는 형태로 바꾼다.
2. 모델 호출이 오래 걸릴 때 점이 움직이는 간단한 애니메이션을 추가한다.
3. 단순 인사와 일반 잡담 판별 범위를 더 넓힌다.

## 핵심 정리

- 단순 인사는 짧게 답하도록 수정했다.
- C++ 학습 답변 형식은 필요한 질문에만 적용한다.
- 응답 생성 중 로딩 메시지를 추가했다.
- 빌드와 기본 실행 확인을 완료했다.
