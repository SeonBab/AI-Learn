# Assistant 작업 및 문서화 운영 규칙

## 설명

이 문서는 사용자가 assistant에게 지시한 내용과 assistant가 수행한 작업 및 답변을 어떻게 문서로 정리할지 정의한다.

이 규칙은 별도의 지시가 있을 때까지 계속 적용한다.

## 예제 코드

```cpp
// 이 문서는 작업 운영 규칙 문서이므로 실행용 C++ 코드는 포함하지 않는다.
```

## 핵심 정리

### 1. AGENTS 파일 수정 제한

`AGENTS.md` 파일은 사용자의 별도 지시가 있을 때까지 더 이상 수정하지 않는다.

문서화 규칙이나 작업 방식이 추가되더라도 `AGENTS.md`에 바로 반영하지 않고, 목적에 맞는 별도 문서를 `docs` 폴더에 작성한다.

### 2. 사용자 지시 기록 규칙

사용자가 새로운 작업을 지시할 때마다 다음 내용을 문서에 정리한다.

1. 사용자가 지시한 내용
2. assistant가 실제로 작업한 내용
3. assistant가 답변한 핵심 내용
4. 생성하거나 수정한 파일
5. 다음에 이어서 할 수 있는 작업

### 3. 문서 분리 규칙

모든 내용을 하나의 문서에 몰아서 정리하지 않는다.

작업 성격에 따라 적절한 문서로 분리한다.

| 작업 성격 | 권장 문서 위치 |
|---|---|
| 프로젝트 코딩 규칙 | `docs/project-conventions.md` |
| 커밋 메시지 규칙 | `docs/commit-conventions.md` |
| assistant 작업 운영 규칙 | `docs/assistant-workflow.md` |
| assistant 답변 기록 요약 | `docs/assistant-response-notes.md` |
| Git 작업 흐름 | `docs/git-workflow.md` |
| C++ 학습 내용 | `docs/cpp-*.md` |
| 문서 목차 | `docs/index.md` |

### 4. 답변 정리 방식

assistant 답변은 그대로 복사하기보다 문서에 맞게 요약하고 구조화한다.

단, 사용자의 지시 문장은 필요한 경우 원문에 가깝게 기록한다.

정리 예시:

```text
요청 내용:
에이전트 파일은 별도의 지시가 있을 때까지 더이상 건드리지마

작업 내용:
- AGENTS.md 수정 제한 규칙을 assistant 작업 운영 문서에 기록했다.
- 이후 작업부터 사용자 지시와 assistant 작업 내용을 문서별로 분리해 정리하도록 정했다.
```

### 5. 이번 지시 기록

요청 내용:

```text
에이전트 파일은 별도의 지시가 있을 때까지 더이상 건드리지마
이제 내가 너한테 지시할 때 마다 내가 지시한 것과 너가 작업한 내용및 답변을 문서로 정리해
문서 하나에 정리하지말고 적절하게 분리하면서 정리해
```

작업 내용:

- `AGENTS.md`를 더 이상 수정하지 않는 규칙을 기록했다.
- 사용자 지시, assistant 작업 내용, 답변 요약을 문서로 정리하는 규칙을 기록했다.
- 모든 기록을 하나의 문서에 몰아넣지 않고 작업 성격에 맞게 분리하는 기준을 정리했다.

관련 문서:

```text
docs/assistant-workflow.md
```
