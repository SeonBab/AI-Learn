# Assistant 작업 및 문서화 운영 규칙

## 설명

이 문서는 사용자가 assistant에게 지시한 내용과 assistant가 수행한 작업을 어떻게 문서로 정리할지 정의한다.

이 규칙은 별도의 지시가 있을 때까지 계속 적용한다.

## 핵심 정리

### 1. AGENTS 파일 수정 기준

`AGENTS.md`는 필요한 경우 수정할 수 있다.

단, 일반 작업 기록을 `AGENTS.md`에 직접 넣지 않는다. 프로젝트 전체에 적용되어야 하는 기준 지침만 `AGENTS.md`에 반영하고, 작업 기록은 목적에 맞는 별도 문서에 정리한다.

### 2. 사용자 지시 기록 규칙

사용자가 새로운 작업을 지시할 때마다 다음 내용을 문서에 정리한다.

1. 사용자가 지시한 내용
2. assistant가 실제로 작업한 내용
3. assistant가 답변한 핵심 내용
4. 생성하거나 수정한 파일
5. 다음에 이어서 할 수 있는 작업

### 3. 문서 분리 규칙

모든 내용을 하나의 문서에 몰아서 정리하지 않는다.

문서 성격에 따라 다음처럼 분리한다.

| 작업 성격 | 권장 문서 위치 |
|---|---|
| 프로젝트 코딩 규칙 | `docs/project-conventions.md` |
| 커밋 메시지 규칙 | `docs/commit-conventions.md` |
| assistant 작업 운영 규칙 | `docs/assistant-workflow.md` |
| assistant 답변 정리 기준 | `docs/assistant-response-notes.md` |
| 개별 작업 기록 | `docs/work-logs/` |
| C++ 학습 내용 | `docs/cpp-*.md` |
| 문서 목차 | `docs/index.md` |

### 4. 작업 기록 폴더 규칙

작업 수행 내용을 문서화할 때는 컨벤션 문서와 섞이지 않도록 `docs/work-logs/` 폴더에 저장한다.

`docs` 루트에는 다음 성격의 문서만 둔다.

1. 프로젝트 기준 문서
2. 코딩 컨벤션
3. 커밋 컨벤션
4. assistant 운영 규칙
5. 문서 목차

개별 작업 기록, 구현 로그, 검토 로그는 `docs/work-logs/` 아래에 작업 성격별 파일로 분리한다.

### 5. 답변 정리 방식

assistant 답변은 그대로 복사하기보다 문서에 맞게 요약하고 구조화한다.

사용자의 지시 문장은 필요한 경우 원문을 가깝게 기록한다.

### 6. 다음 작업 추천

다음에는 `docs/index.md`를 만들어 현재 문서들을 목적별로 연결하는 목차를 구성하는 것을 추천한다.
