# Assistant 답변 정리 기준

## 설명

이 문서는 assistant의 답변과 작업 내용을 프로젝트 문서로 정리할 때 사용할 기준을 설명한다.

답변 정리는 학습 노트로 다시 읽기 쉽도록 요약하고, 필요한 파일 경로와 다음 작업을 함께 남기는 것을 목표로 한다.

## 핵심 정리

### 1. 정리 대상

다음 내용은 문서로 정리한다.

1. 사용자의 주요 지시
2. assistant가 수행한 작업
3. 생성하거나 수정한 파일
4. 답변의 핵심 요약
5. 다음에 이어서 할 수 있는 작업

### 2. 정리하지 않아도 되는 내용

다음 내용은 매번 문서화하지 않아도 된다.

1. 단순한 상태 확인
2. 파일 목록 확인만 수행한 경우
3. 사용자가 문서화를 원하지 않는다고 명시한 경우
4. 코드나 문서에 영향을 주지 않는 짧은 질의응답

### 3. 문서 위치 기준

프로젝트 기준과 관련된 문서는 `docs` 루트에 둔다.

개별 작업 기록은 `docs/work-logs/`에 둔다.

C++ 학습 내용은 필요할 경우 `docs/cpp-*.md` 형식으로 별도 작성한다.

### 4. 문서 작성 형식

문서는 한글 Markdown을 기본으로 작성한다.

C++ 예제가 필요하면 반드시 `cpp` 코드 블록을 사용한다.

```cpp
// C++ 예제 코드는 cpp 코드 블록에 작성한다.
```

### 5. 다음 작업 추천

다음에는 문서 목차인 `docs/index.md`를 만들어 컨벤션 문서와 작업 기록 문서를 구분해서 링크하는 것을 추천한다.
