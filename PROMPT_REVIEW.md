# 프롬프트 검토 기준

## 설명

이 문서는 AI에게 코드 작성을 요청하는 프롬프트가 적절한지 검토하고 개선하기 위한 기준을 정리한다.

목적은 C++ 초보자가 AI에게 코드를 요청할 때 한 번에 너무 많은 기능을 요구하지 않고, 실행, 설명, 검증, 개선까지 확인 가능한 프롬프트를 만드는 것이다.

## 핵심 정리

### 1. 검토 항목

AI가 작성할 코드 요청 프롬프트를 다음 기준으로 검토한다.

1. 한 번에 하나의 기능만 요청하는가?
2. 첫 번째 기능은 실행해서 확인 가능한가?
3. C++ 초보자가 이해할 수 있는 수준인가?
4. 코드 설명을 요구하고 있는가?
5. 실행 방법을 요구하고 있는가?
6. 검증 방법을 요구하고 있는가?
7. 발생 가능한 오류와 주의점을 요구하고 있는가?
8. 개선 방향을 질문하고 있는가?

### 2. AI에게 요청할 검토 프롬프트

아래 프롬프트를 AI에게 제공하여 코드 요청 프롬프트가 적절한지 평가하고 개선하도록 요청한다.

```text
아래 프롬프트 초안을 다음 기준에 따라 검토해줘.

검토 기준:

1. 한 번에 하나의 기능만 요청하는가?
2. 첫 번째 기능은 실행해서 확인 가능한가?
3. C++ 초보자가 이해할 수 있는 수준인가?
4. 코드 설명을 요구하고 있는가?
5. 실행 방법을 요구하고 있는가?
6. 검증 방법을 요구하고 있는가?
7. 발생 가능한 오류와 주의점을 요구하고 있는가?
8. 개선 방향을 질문하고 있는가?

검토할 때는 다음 형식으로 답변해줘.

1. 항목별 평가
2. 부족한 점
3. 수정 방향
4. 개선된 최종 프롬프트

검토 대상 프롬프트:
+[여기에 AI가 작성할 코드 요청 프롬프트를 붙여넣기]
```

### 3. 항목별 평가 작성 방식

항목별 평가는 단순히 좋다 또는 나쁘다로 끝내지 않고 다음 정보를 포함한다.

1. 충족 여부
2. 판단 이유
3. 보완이 필요한 경우의 수정 예시

### 4. 개선된 최종 프롬프트 작성 원칙

개선된 최종 프롬프트는 다음 조건을 만족해야 한다.

1. 기능 범위가 작고 명확해야 한다.
2. C++ 초보자가 따라갈 수 있어야 한다.
3. 실행 방법이 포함되어야 한다.
4. 결과 확인 방법이 포함되어야 한다.
5. 코드 설명을 요구해야 한다.
6. 주의할 점과 다음 개선 방향을 요구해야 한다.

### 5. 최종 프롬프트 예시

```text
C++ 초보자가 이해할 수 있도록 콘솔에서 사용자 이름을 입력받고,
입력받은 이름을 사용해 인사말을 출력하는 프로그램을 작성해줘.

요구사항:

1. 기능은 이름 입력과 인사말 출력만 포함해줘.
2. 전체 코드를 보여줘.
3. 코드의 각 부분이 어떤 역할을 하는지 설명해줘.
4. Visual Studio 또는 g++에서 실행하는 방법을 알려줘.
5. 실행했을 때 어떤 결과가 나와야 하는지 예시를 보여줘.
6. 입력이 비어 있는 경우처럼 주의할 점이 있다면 설명해줘.
7. 이 코드를 다음 단계에서 어떻게 개선할 수 있는지도 알려줘.
```
