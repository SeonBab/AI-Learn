# Assistant 응답 지침

이 프로젝트는 C++ 학습을 돕는 로컬 AI 채팅 assistant를 만들기 위한 프로젝트이다.

모든 assistant 답변은 학습 노트로 저장하기 좋은 Markdown 문서 형식으로 작성한다.

## 필수 답변 구조

1. 제목은 사용자의 질문을 요약한 문장으로 작성한다.
2. 답변은 다음 섹션 구조를 따른다.
   - `## 설명`
   - `## 핵심 정리`
3. C++ 코드가 필요하면 반드시 `cpp` 코드 블록을 사용한다.
4. 답변이 길어질 경우 단계별 목록으로 나누어 작성한다.
5. 불필요한 인사말은 생략한다.

## 언어 규칙

모든 문서 파일은 한글로 작성하는 것을 기본으로 한다.

다만 기술적 정확성, 소스 코드 호환성, 코드 주석, API 이름, 에러 메시지, 공식 용어 유지가 필요한 경우에는 영어를 사용할 수 있다.

사용자가 명시적으로 영어 답변을 요청한 경우에도 영어를 사용할 수 있다.

## 작업 문서화 규칙

사용자가 작업을 지시할 때마다 다음 내용을 문서로 정리한다.

1. 사용자가 지시한 내용
2. assistant가 작업한 내용
3. assistant가 답변한 핵심 내용
4. 생성하거나 수정한 파일
5. 다음에 이어서 할 수 있는 작업

모든 내용을 하나의 문서에 몰아서 정리하지 않고, 작업 성격에 따라 적절한 문서로 분리한다.

## 작업 기록 폴더 규칙

일반 컨벤션 문서와 작업 기록 문서를 섞지 않는다.

작업 지시마다 작성하는 작업 기록 문서는 `docs/work-logs/` 폴더에 저장한다.

`docs` 루트에는 프로젝트 기준, 컨벤션, 운영 규칙처럼 계속 참조되는 문서만 둔다.

`AGENTS.md`는 필요한 경우 수정할 수 있다. 단, 프로젝트 기준 지침에 해당하는 내용만 반영하고 일반 작업 기록은 `docs/work-logs/` 폴더의 목적별 문서에 정리한다.
